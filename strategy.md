# class_cart 전략 문서

> 마지막 업데이트: 2026-03-14

---

## 1. 채널 개요

| 항목 | 내용 |
|------|------|
| 채널명 | **class_cart** |
| 플랫폼 | 쓰레드(Threads), X(Twitter) |
| 포지셔닝 | 클라스 있는 장바구니 — 안목 있는 큐레이터가 직접 고른 제품만 담는다 |
| 수익 모델 | 쿠팡파트너스 (링크 클릭 → 24시간 내 구매 → 수수료) |

---

## 2. 핵심 원칙

1. **광고처럼 보이면 실패다** — 콘텐츠가 먼저, 링크는 자연스럽게 따라온다
2. **"사세요"가 아니라 "공감"이 먼저** — 상황 제시 → 설득 → 솔루션 순서
3. **큐레이터의 안목을 신뢰하게 만든다** — 아무거나 올리지 않는다, 직접 고른 것만

---

## 3. 콘텐츠 공식

```
[후킹] 공감가는 상황/문제 제시
    ↓
[전환] 이럴 때 이런 게 필요하더라
    ↓
[솔루션] 제품 소개 (스펙이 아니라 경험 중심)
    ↓
[링크] 자연스럽게 붙이기
```

### 하지 말 것
- "이거 진짜 좋아요 꼭 사세요"
- 스펙 나열식 소개
- 링크만 던지는 포스트
- 모든 포스트에 링크 넣기

### 해야 할 것
- "이런 경험 있지 않아?" 로 시작
- 내가 왜 이걸 골랐는지 이유 설명
- 링크 없는 순수 콘텐츠도 섞기 (신뢰 구축)
- 댓글/반응에 적극 소통

### 캐러셀 이미지 구성 (4장 기본)

| 슬라이드 | 역할 | 핵심 |
|---------|------|------|
| 1 - Hook | 시선 잡기 | 다크 배경 + 후킹 헤드라인 + 카테고리 뱃지 |
| 2 - Product | 제품 소개 | 클린 배경 + 제품 사진 + 핵심 장점 |
| 3 - Lifestyle | 공감/신뢰 | 사용 장면 + 후기 인용문 |
| 4 - CTA | 행동 유도 | "댓글에 링크 남겨뒀어" + 브랜딩 |

- 비율: 4:5 세로형 (1080x1350px)
- 캐러셀은 텍스트 전용 대비 최대 596% 인게이지먼트 상승
- 매 슬라이드가 독립적 가치를 제공하면서 다음 슬라이드 기대를 만들 것
- 링크 포스트는 전체 게시물의 약 20% (5개 중 1개), 나머지는 순수 콘텐츠

## 4. 플랫폼별 전략

### 쓰레드 (Threads) — 메인 플랫폼
- 텍스트 + 캐러셀(4~5장) 조합이 기본 포맷
- 일상 대화체
- 해시태그 적극 활용

#### 링크 배치 전략
- **본문에 링크 넣지 않는다** (외부 링크 포함 시 노출 대폭 감소)
- **댓글에 링크 작성 → 고정댓글로 고정**
- 게시 직후 30분간 댓글 응대에 집중 (알고리즘 초기 부스트)
- 프로필 링크 슬롯 5개 활용 (링크허브 또는 주요 추천 링크)
- 경제적 이해관계(쿠팡파트너스) 반드시 표시

#### 게시 흐름
1. 본문 게시 (링크 없이, 콘텐츠만)
2. 초기 노출 확보 확인
3. 댓글로 쿠팡파트너스 링크 추가 → 고정
4. 30분간 댓글 응대 집중

### X (Twitter) — 크로스포스팅
- 쓰레드 콘텐츠를 동일하게 게시 (우선)
- 짧고 임팩트 있는 후킹
- 스레드(연속 트윗)로 스토리텔링
- 인용 트윗으로 기존 화제에 올라타기
- 트렌드/시즌 이슈 빠른 대응

## 5. 콘텐츠 카테고리

| 카테고리 | 설명 | 예시 |
|----------|------|------|
| 생활 꿀템 | 일상에서 삶의 질 올려주는 제품 | 정리용품, 주방도구, 청소용품 |
| 시즌/날씨 | 계절·날씨에 맞는 타이밍 제품 | 여름 냉감템, 겨울 보온템, 장마 대비 |
| 문제 해결 | 누구나 겪는 불편함에 대한 솔루션 | 수납 고민, 수면 문제, 자세 교정 |
| 트렌드 | 지금 뜨는 것, 화제인 것 | SNS 바이럴 제품, 신제품 |
| 가성비 | 가격 대비 성능 좋은 발견 | "이 가격에 이 퀄리티?" |

## 6. 콘텐츠 제작 및 업로드

### 제작 파이프라인

```
[1] 상황 분석 → [2] 제품 선정 → [3] 콘텐츠 제작 → [4] 검수 → [5] 게시
```

#### [1] 상황 분석 (상황이 먼저, 제품은 나중)
5가지 축을 입체적으로 분석:
- **계절/날씨**: 환절기, 장마, 한파 등 + 이번 주 날씨
- **시기/이벤트**: 명절, 새학기, 기념일, 연휴
- **트렌드/바이럴**: SNS 화제 제품, 검색 트렌드
- **신제품/출시**: 브랜드 신제품, 리뉴얼
- **사회 이슈**: 건강 이슈, 경제 트렌드, 라이프스타일 변화

→ "지금 이 시점에 가장 공감될 상황"을 먼저 설계

#### [2] 제품 선정
- 설계된 상황에 맞는 키워드로 쿠팡파트너스 API 검색
- 선정 기준: 스토리 가능성 > 가격 접근성(만원~5만원) > 시의성 > 로켓배송
- 최근 다룬 카테고리/패턴 겹침 방지

#### [3] 콘텐츠 제작
- 6가지 심리학 패턴 중 카테고리에 맞는 패턴 선택 → 텍스트 생성
- Pencil(.pen)로 캐러셀 4장 디자인

#### [4] 검수
- 텍스트 체크리스트 검증 (30자 후킹, 500자 이내, CTA 자연스러움)
- 캐러셀 스크린샷 확인

#### [5] 게시
- API 자동 게시 → 댓글에 링크 → 고정

### 쿠팡파트너스 API
- 가이드 문서: https://partners.coupang.com/#help/open-api
- 주요 엔드포인트: 상품 검색, 골드박스(핫딜), 카테고리 베스트, 딥링크 생성
- 검색 API 제한: **10회/시간** (DB 캐싱 필수)
- 응답 데이터: 상품명, 가격, 이미지URL, 로켓배송 여부, 제휴 링크
- 403 에러 3회 연속 시 영구 차단 → 반드시 백오프 적용

### 콘텐츠 심리학 패턴 (6가지 로테이션)

| 패턴 | 심리 원리 | 최적 카테고리 |
|------|-----------|-------------|
| 반전 고백형 | 부정→반전 = 설득력 상승 | 건강식품, 가전 |
| 일상 침투형 | "이거 나인데?" = 방어막 해제 | 뷰티, 생활, 식품 |
| 비교 파괴형 | 현명한 소비의 쾌감 | 가전, 뷰티 |
| 조용한 경고형 | 손실 회피 본능 자극 | 건강, 위생 |
| 선물 추천형 | 타인 위한 소비 = 합리화 쉬움 | 리빙, 가전 |
| 발견 공유형 | 독점 정보의 우월감 | 생활, 식품 |

- 같은 패턴 연속 사용 금지 (6가지 돌려가며)
- 5개 게시물 중 1개만 제품 링크, 나머지는 순수 콘텐츠

### 게시 자동화 (API)

| | Threads API | X API (Free) |
|--|-------------|-------------|
| 비용 | 무료 | 무료 |
| 게시 제한 | 24시간 250개 | 월 1,500개 |
| 링크 | `link_attachment` 파라미터 | 자동 생성 |
| 인증 | OAuth 2.0 (토큰 60일) | OAuth 2.0 / 1.0a |
| 심사 | Meta 앱 심사 필요 | 비교적 간단 |

#### Threads API 게시 프로세스
1. `POST /{user-id}/threads` — 컨테이너 생성
2. 30초 대기
3. `POST /{user-id}/threads_publish` — 발행
4. 댓글로 링크 추가 (별도 API 호출)

#### 필요 권한
- `threads_basic` — 기본 접근
- `threads_content_publish` — 게시
- `threads_manage_replies` — 댓글 관리 (링크 댓글용)
- `threads_manage_insights` — 성과 분석
